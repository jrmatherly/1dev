## Why

Running `bun run dev` without `MAIN_VITE_DEV_BYPASS_AUTH=true` renders a working login screen, but clicking **Sign in** opens the dead upstream URL `https://apollosai.dev/auth/desktop?auto=true`. The MSAL (Entra ID) sign-in path was wired into `auth-manager.ts` by the archived `wire-enterprise-auth` change (2026-04-09) and is fully implemented — but the `enterpriseAuthEnabled` feature flag defaults to `false`, so the legacy SaaS branch fires instead. Both Entra app registrations (`LLMOps - 1Code API` `52d25f5d-...` and `LLMOps - 1Code API Graph Client` `0065f604-...`) are provisioned in the tenant; cluster manifests are deployed (`onecode_api_enabled: true` in `cluster.yaml`); the desktop code is wired. The remaining gap is one flag flip plus two env vars plus removal of the dead-URL fallthrough. Doing this now unblocks daily development without dev-bypass and prevents the dead-URL UX foot-gun from regressing.

Full investigation findings, evidence, and cross-reference map: [`docs/enterprise/auth-login-button-wire-msal.md`](../../../docs/enterprise/auth-login-button-wire-msal.md).
